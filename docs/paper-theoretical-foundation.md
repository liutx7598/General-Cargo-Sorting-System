## 2. 研究背景与问题提出

### 2.1 件杂货配载问题的行业背景

件杂货运输对象通常包括钢材、木材、设备箱、工程件、危险品包装货、大型异形件等，其共同特征是：

- 货物尺寸非标准化
- 单件重量差异大
- 可旋转性和可堆叠性差异显著
- 存在危险品隔离、忌装、通风、安全距离等复杂规则
- 配载不仅影响舱容利用，还直接影响整船重心、稳性与运营安全

因此，件杂货配载不是单纯的“体积装箱”问题，而是一个同时包含：

- 几何排布问题
- 组合优化问题
- 船舶稳性问题
- 规则约束推理问题
- 可视化决策支持问题

的复杂综合优化问题。

### 2.2 件杂货配载与集装箱配载的差异

从研究视角看，件杂货配载与集装箱配载有本质差异：

1. 集装箱相对标准化，件杂货具有高度异构性。
2. 集装箱更多关注舱位分配、港序和重量分布；件杂货更强调舱内真实几何摆位。
3. 件杂货中“同层成排、同舱成块、先装满一个舱再考虑下一舱”是一种强业务规律。
4. 危险货、异形货、不可叠货、偏心货物对几何和稳性影响更直接。
5. 件杂货需要更细粒度的可解释结果，以便配载员审核和人工修正。

因此，本项目的学术意义在于：针对件杂货场景构建一套“几何计算 + 稳性核算 + 规则推理 + 混合优化 + 可视化解释”的统一方法框架。

### 2.3 研究问题

本系统可以对应为如下研究问题：

给定一艘船的多货舱结构、一批具有尺寸、重量、类别、危险属性、偏心属性和堆叠约束的货物，以及若干稳性和安全规则，如何构建一个既满足空间与规则约束、又兼顾稳性与舱位组织效率、并能够生成可解释配载图的智能配载方案？

这一问题可形式化为一个多目标、多约束、混合离散优化问题。

## 3. 理论基础总览

本系统的理论基础主要由五部分构成：

1. 船舶与货物三维几何建模理论
2. 质量中心与稳性计算理论
3. 组合优化与三维装箱理论
4. 规则约束与安全隔离理论
5. 决策支持系统与可视化表达理论

这五部分并不是孤立存在，而是通过“几何 -> 重心 -> 稳性 -> 规则 -> 结论”的依赖链联结为一体。

## 4. 理论基础一：三维几何建模与空间表示

### 4.1 坐标系定义

系统采用船舶三维坐标系：

- `x` 轴：船尾到船首
- `y` 轴：左舷到右舷
- `z` 轴：基线向上

这一坐标系的意义在于：

- `x` 方向直接影响纵向重心 `LCG`
- `y` 方向直接影响横向重心 `TCG`
- `z` 方向直接影响垂向重心 `KG`

因此，几何坐标不仅服务于摆位显示，更直接进入稳性计算。

### 4.2 货物几何表示

每件货物以轴对齐长方体近似表示，其原始尺寸为：

- 长：`l_i`
- 宽：`b_i`
- 高：`h_i`

研究上，这相当于将真实货物抽象为最小包围盒。该抽象有三点意义：

1. 它使复杂不规则货物可以进入统一的优化框架。
2. 它允许使用三维装箱问题中的经典边界与碰撞判断。
3. 它与实际港航作业中的“外形尺寸录单”高度一致，具有工程合理性。

### 4.3 货物朝向与离散旋转

系统支持六种正交摆放方式：

- `LWH`
- `LHW`
- `WLH`
- `WHL`
- `HLW`
- `HWL`

本质上，这是对长、宽、高三个维度的排列组合。其理论基础是：

- 对于轴对齐长方体，若忽略任意角度倾斜，仅考虑正交旋转，则可行朝向集合为 3! = 6 种。
- 这种离散旋转比连续旋转更适合工程系统，也更符合现实作业中的吊装与摆放方式。

旋转后的尺寸记为：

- `a_i`
- `c_i`
- `d_i`

对应关系为：

`(a_i, c_i, d_i) = rotate_dimensions(l_i, b_i, h_i, orientation_i)`

### 4.4 舱内原点与摆位表示

若货物左后下角原点为：

- `(x0_i, y0_i, z0_i)`

则一件货物的空间占据范围为：

- `x ∈ [x0_i, x0_i + a_i]`
- `y ∈ [y0_i, y0_i + c_i]`
- `z ∈ [z0_i, z0_i + d_i]`

从研究角度看，这使问题转化为“带有方向选择的三维矩形体排布问题”。

## 5. 理论基础二：重心、力矩与稳性理论

### 5.1 单件货物重心理论

在最简单情况下，轴对齐长方体的几何中心即其重心。若考虑偏心，则需引入中心偏移量：

- `dx_i = centerOffsetX`
- `dy_i = centerOffsetY`
- `dz_i = centerOffsetZ`

于是货物重心为：

- `x_i = x0_i + a_i / 2 + dx_i`
- `y_i = y0_i + c_i / 2 + dy_i`
- `z_i = z0_i + d_i / 2 + dz_i`

这一模型的学术意义在于：

- 它把“几何摆放”转换成“质量分布”
- 它允许系统处理偏心设备、大型机械、包装不均匀货等实际情形
- 它是后续货舱合重心与整船重心的基础

### 5.2 单件货物体积

体积定义为：

- `V_i = a_i * c_i * d_i`

体积的用途有两类：

1. 用于舱容利用率计算
2. 用于分舱阶段的容量约束和近似装载评分

### 5.3 货舱合重心理论

设第 `n` 个货舱货物集合为 `H_n`，则总重量：

- `W_n = Σ W_i`

货舱合重心：

- `X_n = Σ(W_i * x_i) / W_n`
- `Y_n = Σ(W_i * y_i) / W_n`
- `Z_n = Σ(W_i * z_i) / W_n`

这是典型的离散质点系重心公式，其理论意义是：

- 将多货物舱内分布压缩为可解释的统计指标
- 为舱间平衡与局部受力分析提供基础
- 为结果页中每舱指标卡提供定量支撑

### 5.4 整船总排水量理论

当前 MVP 版本采用简化排水量模型：

- `Δ = W0 + ΣW_i`

其中：

- `W0` 为空船重量
- `ΣW_i` 为全部货物重量

系统还预留了以下扩展项：

- `fuelWeight`
- `ballastWeight`
- `freshWaterWeight`
- `storesWeight`

扩展后：

- `Δ = W0 + ΣW_i + fuelWeight + ballastWeight + freshWaterWeight + storesWeight`

这一设计符合船舶稳性计算的标准框架，具有良好的扩展性。

### 5.5 整船总重心理论

整船纵向重心：

- `X_g = (W0 * Xg0 + Σ(W_i * x_i) + otherLongMoment) / Δ`

整船横向重心：

- `Y_g = (W0 * Yg0 + Σ(W_i * y_i) + otherTranMoment) / Δ`

整船垂向重心：

- `KG = (W0 * KG0 + Σ(W_i * z_i) + otherVertMoment) / Δ`

这里本质上依然是质量与力矩平衡原理。研究意义在于：

- 将局部摆位结果统一投影到整船稳性指标
- 说明“舱内排布”与“船舶安全”之间不是经验关系，而是明确的力学关系

### 5.6 纵横垂三向力矩

系统显式输出三类力矩：

- `longitudinalMoment = Σ(W_i * x_i)`
- `transverseMoment = Σ(W_i * y_i)`
- `verticalMoment = Σ(W_i * z_i)`

这些量有三个作用：

1. 解释总重心计算过程
2. 为后续扩展剪力、弯矩、压载联动提供基础
3. 增强结果可解释性，使系统不只是给出结论，还能展示推导过程

### 5.7 GM 稳性理论

系统采用简化初稳性模型：

- `GM = KM(Δ) - KG - FSC`

其中：

- `KM(Δ)` 由静水力表按排水量插值得到
- `KG` 为整船垂向重心
- `FSC` 为自由液面修正，当前默认 `0`

静水力表插值原理为线性插值：

若 `Δ` 位于 `Δ1` 与 `Δ2` 之间，则：

- `KM(Δ) = KM1 + ((Δ - Δ1) / (Δ2 - Δ1)) * (KM2 - KM1)`

这一模型的学术意义是：

- 保留了初稳性分析的核心物理含义
- 既满足 MVP 阶段的工程可行性，又保留向高保真稳性模型扩展的路径

### 5.8 GM 变化量

定义：

- `deltaGM = GM_loaded - GM_reference`

当前系统以设计 GM 作为参考：

- `GM_reference = designGM`

该指标用于说明装载后稳性相对空船或设计状态的变化趋势。

### 5.9 舱容利用率与载重密度

货舱体积利用率：

- `η_n = ΣV_i / V_n`

单位舱容载重：

- `λ_n = W_n / V_n`

相邻货舱利用率差异：

- `R_n = |η_n - η_(n+1)|`

这些指标用于衡量：

- 空间利用程度
- 装载是否过度分散
- 相邻货舱负载是否均衡

### 5.10 纵向集中指标

系统定义简化指标：

- `I_x = Σ(W_i * |x_i - X_ref|)`

其中 `X_ref` 可取船中或当前总纵向重心。当前实现中使用当前 `LCG`。

这一指标反映货物是否过分集中于某一纵向区段。其意义类似于将重量分布相对于参考位置的离散程度量化。

## 6. 理论基础三：组合优化与三维装箱理论

### 6.1 问题本质

本系统解决的问题可以拆为两个相互耦合的子问题：

1. 货物到货舱的分配问题
2. 已分配货物在货舱内的三维摆位问题

前者属于带容量、兼容性和多目标权衡的分配优化问题；后者属于带朝向、层位、碰撞与隔离约束的三维装箱问题。

从理论上看，这类问题属于 NP-hard 问题家族，不适合用单一精确算法在工程时限内全局穷举。因此系统采用“启发式 + 精确求解 + 局部修复”的混合框架。

### 6.2 为什么不能只用单一精确优化

若把所有决策变量一次性放入统一 MIP/CP 模型，会面临：

- 变量维度爆炸
- 非重叠约束数量呈平方增长
- 旋转与层位选择带来离散组合爆炸
- 工程响应时间难以满足交互系统要求

因此，项目采用分阶段求解：

- 先用静态筛选减少候选空间
- 再做分舱优化
- 再做单舱局部摆位
- 最后做核算与回退修复

这种分而治之的策略在工程智能优化系统中具有很强合理性。

### 6.3 静态筛选的理论依据

静态筛选阶段的目标是减少组合空间。其理论基础是“先验可行域缩减”。

对每件货物先筛掉：

- 尺寸无论如何旋转都放不下的货舱
- 重量超出舱限的货舱
- 明确不可装载的货舱

得到 `candidateHolds`。该步骤不会求最优解，但会显著降低后续优化复杂度。

### 6.4 分舱阶段的理论模型

设：

- `x_ih ∈ {0,1}` 表示货物 `i` 是否分配到货舱 `h`
- `u_h ∈ {0,1}` 表示货舱 `h` 是否被开启

基本约束包括：

1. 唯一分配约束  
   `Σ_h x_ih = 1`

2. 货舱开启联动  
   `x_ih <= u_h`

3. 重量容量约束  
   `Σ_i W_i * x_ih <= W_h^max`

4. 体积容量约束  
   `Σ_i V_i * x_ih <= V_h`

5. 不兼容货物分离约束  
   若 `i,j` 不兼容，则 `x_ih + x_jh <= 1`

### 6.5 当前分舱目标函数的理论解释

项目当前的 CP-SAT 目标函数具有多目标加权性质，可概括为：

最小化：

- 开舱数量
- 开启低优先级货舱的代价
- 货物偏离中部优先舱的代价
- 货舱利用率偏离平均水平的代价
- 与贪心种子解偏差的代价

从运筹学角度，这是将多目标问题转化为加权单目标问题。其合理性在于：

- “最少开舱”反映件杂货的实际作业偏好
- “靠近船中”有助于降低纵向偏载风险
- “利用率适度均衡”避免极端分布
- “贪心种子偏置”有助于提高求解稳定性和速度

### 6.6 件杂货特化：优先装满一个舱

本系统的一个关键场景化理论点是：

件杂货作业往往优先形成“成舱、成块、成排”的装载格局，而不是像某些平均分布模型那样把货物平均摊到各舱。

因此，系统在分舱阶段引入了强烈的“少开舱偏好”，包括：

- 高权重惩罚开启新舱
- 已开启舱优先继续装载
- 中部舱优先于边缘舱

这使优化目标不再是抽象均衡，而是面向真实件杂货作业逻辑的“紧凑装舱”。

### 6.7 舱内摆位的理论基础

舱内摆位采用分层 + 分排 + Best-Fit Decreasing 风格启发式，其理论基础来自二维/三维装箱中的货架式装箱思想（shelf packing）。

核心结构：

- `Layer`：垂向货层
- `Shelf`：同一层中的横向货排

其好处是：

- 更符合件杂货平铺、成排的人工操作方式
- 比任意三维自由摆放更可解释
- 便于后续生成二维总配载图
- 更适合与“不可叠货”“优先同层聚拢”等规则结合

### 6.8 舱内摆位评分函数的理论解释

当前启发式对候选摆位进行打分，核心思想是优先：

- 低层优先
- 低高度膨胀优先
- 平放优先
- 已有货排优先
- 同层新排优先于新开层
- 剩余空间较小且稳定脚印较大的方案优先

这意味着系统不是单纯为了“能放下”而放，而是试图逼近更稳、更规整、更符合件杂货经验的摆位方式。

### 6.9 为什么采用“层优先 + 同层聚拢”

这一策略的理论合理性在于：

1. 低层装载有助于降低 `KG`
2. 同层聚拢有助于减少局部空隙和无谓立放
3. 新开层会提高垂向重心，因此应被惩罚
4. 件杂货现场通常偏好“铺层”而不是早早堆高

因此，该启发式不仅是几何策略，也是稳性导向策略。

### 6.10 回退与局部修复

由于启发式和分阶段求解不能保证一步成功，系统引入回退机制：

- 若某些货物在已分配舱内无法摆入
- 则将未摆入货物重新送回分舱修复流程
- 通过换舱、调层、再摆位完成修复

这是一种典型的局部邻域搜索与迭代修复思想，可视为工程可用性的关键保证。

## 7. 理论基础四：规则约束与安全校验

### 7.1 边界约束理论

若货物属于第 `n` 个货舱，则必须满足：

- `0 <= x0_i`
- `x0_i + a_i <= L_n`
- `0 <= y0_i`
- `y0_i + c_i <= B_n`
- `0 <= z0_i`
- `z0_i + d_i <= H_n`

其本质是矩形体必须完全包含于货舱边界内。

### 7.2 不重叠约束理论

同一货舱内任意两件货物 `i, j` 至少要在一个方向上分离：

- `x0_i + a_i <= x0_j`
- 或 `x0_j + a_j <= x0_i`
- 或 `y0_i + c_i <= y0_j`
- 或 `y0_j + c_j <= y0_i`
- 或 `z0_i + d_i <= z0_j`
- 或 `z0_j + d_j <= z0_i`

这是轴对齐长方体不相交的标准判定条件。

### 7.3 安全间距理论

若货物 `i, j` 需要最小隔离距离 `D_ij`，则用包围盒最短距离表示：

- `delta_x = max(0, x0_j - (x0_i + a_i), x0_i - (x0_j + a_j))`
- `delta_y = max(0, y0_j - (y0_i + c_i), y0_i - (y0_j + c_j))`
- `delta_z = max(0, z0_j - (z0_i + d_i), z0_i - (z0_j + d_j))`
- `Dist(i,j) = sqrt(delta_x^2 + delta_y^2 + delta_z^2)`

约束为：

- `Dist(i,j) >= D_ij`

### 7.4 不兼容与危险货规则的理论表达

系统将规则拆成两类：

1. 不兼容标签规则  
   若货物有冲突标签交集，则视为不兼容。

2. 危险货隔离规则  
   若两者均为危险货，则至少满足默认隔离距离。

该设计体现了一个重要理论观点：

隔离约束是“货对级”的，而不是“单货物级”的全局刚性距离。也就是说，单件货物存在隔离等级，并不意味着它必须与所有其他货物都保持距离，只有形成危险或不兼容配对时才触发。

这一点对减少误报、提升规则合理性非常关键。

### 7.5 规则校验与可解释性

系统把规则结果输出为结构化告警：

- `BOUNDARY`
- `OVERLAP`
- `INCOMPATIBLE`
- `ISOLATION`
- `UNPLACED`

这体现了规则引擎的可解释设计思想：  
系统不仅给出“失败”，还给出失败类型、货物对象和具体文本说明。

## 8. 理论基础五：综合判定与决策支持

### 8.1 最终合规性判定

若同时满足：

- 无空间越界
- 无碰撞
- 无忌装违规
- 无隔离违规
- `GM >= GM_min`
- 舱容比在阈值内
- 相邻舱差异在阈值内
- `Ix <= Ix_max`

则：

- `complianceStatus = PASS`

否则：

- `complianceStatus = FAIL`

该规则相当于一个多条件逻辑合取模型。其本质是把几何、规则和稳性结果汇总到统一决策层。

### 8.2 为什么需要可视化

件杂货配载不是黑箱系统可以单独完成的任务，实际业务中仍然需要配载员复核。因此系统采用：

- 二维总配载图
- 单舱 3D 检视
- 指标卡 + 图表 + 告警表

构成多视角决策支持界面。

其理论意义在于：

- 可解释人工智能
- 人机协同决策
- 降低复杂空间解对用户的理解门槛

## 10. 当前系统中的核心假设

为保证 MVP 可运行性，当前系统采用以下研究假设：

1. 货物近似为轴对齐长方体。
2. 仅考虑六种正交离散朝向，不考虑连续倾斜。
3. 货舱内部不考虑复杂结构遮挡、舱口限制、绑扎点等微观障碍。
4. GM 采用简化初稳性模型，不进行完整静稳性曲线计算。
5. 当前排水量主要考虑空船与货物，燃油、压载、水舱等作为扩展项预留。
6. 舱内摆位以“规则可行 + 工程合理 + 可解释”为主，不追求全局绝对最优。

这些假设不会使研究失去价值，反而体现了工程系统研究中常见的“先建有效近似模型，再逐步扩展高保真能力”的方法路径。

## 12. 论文可提炼的创新点

以下内容可作为论文中的创新点或贡献点候选：

### 12.1 理论与工程一体化建模

系统将三维几何、质量重心、GM 稳性、规则校验与可视化表达统一在同一个数据链路中，避免了“几何一套、稳性一套、展示一套”的割裂问题。

### 12.2 面向件杂货场景的特化优化策略

与强调均匀分布的泛化模型不同，系统明确引入了件杂货领域偏好：

- 优先成舱
- 优先同层聚拢
- 优先平放与低重心摆位

- 启发式舱内摆位
- 局部修复与回退

这种混合框架兼顾了解质量、求解速度与工程稳定性。

- 失败原因列表
- 告警类型
- 各舱指标
- 重心、GM、利用率等中间量
- 二维总配载图和单舱 3D 检视

这为“可解释智能决策系统”提供了实际案例。

然后比较：

- 可行率
- 开舱数
- GM
- 求解时间
- 告警数

这样可以显著增强论文方法论证力度。
